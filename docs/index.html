GitOps Pipeline

ArgoCD Sync Waves

The entire cluster is bootstrapped from a single root Application. Sync waves enforce dependency order across all components.

W0

Vault + External Secrets

Foundation layer. Vault provides secrets, ESO syncs them to K8s. Everything else depends on this.

W1

Istio Base CRDs

Custom Resource Definitions for the service mesh. Required before istiod can start.

W2

Istiod Control Plane

Ambient profile control plane. Manages mesh configuration and certificate distribution.

W3

Istio CNI + ztunnel

Data plane components. CNI plugin intercepts traffic; ztunnel provides transparent mTLS.

W4

kagent + AgentGateway CRDs

Custom Resource Definitions for the AI agent framework and gateway.

W5

kagent + AgentGateway Apps

The AI agent controller, tool server, UI, and gateway instances.

bootstrap/root-app.yaml 
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: root-app
  namespace: argocd
spec:
  source:
    repoURL: https://github.com/ProfessorSeb/k8s-iceman.git
    path: apps
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
Bootstrap Process
  1. Create argocd namespace
  2. Install Argo CD v2.14.11
  3. Wait for deployments to be ready
  4. Install Gateway API CRDs v1.4.0
  5. Apply root App of Apps
  6. Run vault-init.sh post-deploy
Vault Bootstrap
  1. Initialize Vault (1 key share / 1 threshold)
  2. Unseal Vault
  3. Enable KV v2 engine at secret/
  4. Configure Kubernetes auth for ESO
  5. Create policies + roles
  6. Store secrets: LLM API keys, Slack creds, GitHub PAT, F5 credentials
Multi-Source Helm Pattern
Each ArgoCD Application uses multi-source: the Helm chart is pulled from its registry (by version), and the values file comes from this Git repo under helm-values/. This keeps chart upgrades atomic and version-pinned.

Protocol Deep Dive

Agent-to-Agent (A2A)

How bots communicate with kagent agents using JSON-RPC 2.0 over HTTP.

Send a message to an agent 
POST /api/a2a/kagent/telegram-k8s-agent/
Content-Type: application/json

{
  "jsonrpc": "2.0",
  "id": "msg-001",
  "method": "message/send",
  "params": {
    "message": {
      "role": "user",
      "messageId": "abc-123",
      "contextId": "user-42",
      "parts": [
        { "kind": "text",
          "text": "List all pods in kagent" }
      ]
    }
  }
}

Key Protocol Details

  • method — Always message/send
  • parts[].kind — Use kind, not type
  • contextId — Per-user session tracking
  • URL — Trailing slash required!

HITL Approval Flow

  1. Agent requests tool approval
  2. Bot renders approve/reject buttons
  3. User clicks a button
  4. Bot sends decision via A2A DataPart
  5. Agent proceeds or aborts
